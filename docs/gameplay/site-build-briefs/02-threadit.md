# Site brief 02: ThreadIt

## Core promise

ThreadIt teaches that votes and confidence can rank attention, but they do not
prove a claim. Finn repairs reply order, source lineage, and duplicate-claim
detection.

## Runtime identity

| Element | Direction |
| --- | --- |
| Archetype | Forum and link-aggregation thread |
| Visual language | Pale gray cards, charcoal text, orange accents, nested connector lines |
| Bad rule | `MOST VOTES WINS REALITY` |
| Repaired rule | `VOTES RANK ATTENTION, NOT TRUTH` |
| Progress fiction | Reply/source tree reconnects branch by branch |
| Midpoint | Consensus Cascade |
| Secured label | `SOURCE TREE STABLE` |

Use an original two-speech-bubble knot mark. Do not use Reddit's alien, exact
logo, or trade dress.

## Layout

Default Recovery Browser composition:

- left rail: thread title, original question, sort controls;
- center: nested replies with broken connector lines;
- right rail: source tree and duplicate-origin panel;
- bottom status strip: active source trace and AI write log.

The Reading Companion sits outside the thread. Do not place the passage inside a
forum post, comment editor, or reply box.

## Three-act flow

### Act I: Untangle

Corrupted thread:

```text
MOST VOTES WINS REALITY

The highest number is the truest answer. Replies may appear before questions if
they are popular enough.
```

Each accepted reading result restores one or more visible relationships:

- original question returns to the top;
- reply order becomes chronological within a branch;
- a citation reconnects to the post that introduced it;
- duplicate replies collapse into a single shared origin.

There is no percentage meter. Progress is the source tree becoming less tangled.

### Midpoint: Consensus Cascade

The highest-voted answer replicates across every branch. The page briefly looks
more orderly, then becomes suspicious because every branch now says the same
thing.

Visible system copy:

```text
CONSENSUS AUTO-FIX AI has summarized the thread.

Problem: the summary is now being copied into every branch as proof that the
summary was correct.
```

Amy line:

```text
Those are not ten sources. They are one post wearing ten hats.
```

Chinmay line:

```text
Technically, the AI found agreement extremely efficiently.

I am now being told that creating the agreement first is considered a problem.
```

### Resolution: Source Tree Stable

Finn restores independent source lineages and marks the generated consensus
posts as duplicates.

Payoff:

```text
SOURCE TREE STABLE

Original question: restored.
Independent sources: separated.
Duplicate-source branches: quarantined.
```

Blocked write:

```text
ConsensusHelper_4 tried to post the same claim again.

POSTING PAUSED: DUPLICATE SOURCE
```

Evidence file:

```text
THREADIT / SYNTHETIC CONSENSUS OVERFLOW

Votes measured attention. One AI-generated summary copied itself into many
branches and pretended to be independent agreement.
```

## Character states

- Early Amy: `amy_neutral` or `amy_tools`.
- Midpoint Amy: `amy_evidence`.
- Chinmay: `chinmay_confident` early if this is an early campaign site;
  `chinmay_fluster_1` if the global story has reached background writes.
- Techno: `techno_clue_point` or `techno_alert_ball_pin`; her ball rolls along
  the one connector shared by the cloned branches.

## Reading lane

Use argumentative nonfiction, paired claims, and evidence-based disagreement.
The current sampler is `Why Disagreement Matters`.

Future deck direction:

- public-domain argument and rhetoric adapted for grades 10-12;
- original teen-relevant debates with clear claims and counterclaims;
- science and history passages that distinguish evidence from consensus.

Avoid copying real forum posts.

## Acceptance checks

- Can the player see the original question separate from replies?
- Does progress repair the thread structure rather than fill a generic bar?
- Does the midpoint show one source pretending to be many?
- Does the ending preserve disagreement while blocking duplicate-source spam?
- Does the site avoid treating popularity as proof?
