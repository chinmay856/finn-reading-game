# Original parody-site mark manifest

These production-approved SVG marks replace real-product logos and generated
concept-board marks in runtime UI. They are wrapper assets.

| Site | Asset ID | File | Construction |
| --- | --- | --- | --- |
| WikiWhy | `wikiwhy.mark.globe` | `wikiwhy-mark.svg` | latitude/longitude globe plus evidence/question motif |
| ThreadIt | `threadit.mark` | [`../threadit/threadit-mark.svg`](../threadit/threadit-mark.svg) | two interlocked conversation cards |
| FacePlace | `faceplace.mark` | `faceplace-mark.svg` | overlapping profile frames |
| MyCorner | `mycorner.mark` | `mycorner-mark.svg` | nested personalized page corners |

| ViewTube | `viewtube.mark` | `viewtube-mark.svg` | CRT with an angular play bolt |

| Spotty-Fi | `spottyfi.mark` | `spottyfi-mark.svg` | offset cassette and signal dial |
| MapGuess | `mapguess.mark` | `mapguess-mark.svg` | folded map with a question-pin |

All SVGs use a `128 128` view box and transparent background. Pair them with
visible site-name text. The mark alone is not the accessible name.

Do not alter these into a real platform's letter mark, mascot, smile/arrow,
streaming-wave icon, play rectangle, pin, wordmark, exact color order, or trade
dress. Familiarity comes from archetype, parody name, layout role, and the
site-specific palette defined in its build brief.
